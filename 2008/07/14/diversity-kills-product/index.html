多歧亡羊？

Jul 14th, 2008 | Comments

试用 KDE 中，GTK 类的桌面系统这段时间不想去理它们了。原因说来自己都觉得可笑，因为 Firefox 3 的字符显示突然出了问题，把空格一律当作制表符显示。而且是不管是程序界面还是网页内容，一律平等对待，正让我很纠结。初步的摸索结果是 libpango 的问题，这是 firefox 依赖的库，用以处理 i18n 的字符显示，依稀记得我在编译安装 tint 的过程中有装过类似的包，然而早就忘记了，而跑 Synaptic 里头看历史记录自然是看不到的。多嘴说一下，aptitude install 的 tint 居然是个俄罗斯方块，囧。

问题解决不了，firefox 又是每天必用程序，心中某种情绪作祟之下，我决定删掉 firefox-3.0，降级到 firefox-2。如此之后居然还不满足，于是又装了 KDE。进来之后的第一反应是，KDE4 确实相当华丽。具体怎么个华丽法，可以看看风骚的开始菜单。然而惊喜不足半小时，Konqueror 不幸挂掉一次。因为每次体验（从来没有长期使用过）KDE 的时候用的版本都非常的新，我喜欢形容这种挂掉为故态复萌……

有的时候我自己都很好奇，GNU/Linux 的搭配方案是如此之多，问题的解决之道往往大相径庭，会不会因为这一特征造成的使用者分散（看看桌面环境的选择，套餐有 GNOME、KDE、XFCE 等等，自选有 Ion、WMII、FVWM、Sawfish、Openbox 等等等等）、统一的技术支持困难（去下显卡驱动？您是要 src 包自己编译安装阿还是要打包好的 rpm 安装包或者源码包阿还是 deb 安装包或者源码包阿？哦，您用的是 arch/gentoo/freebsd？应该也有支持的吧……）等等的不确定性而渐渐走向消亡？

我觉得我是吃饱了撑的，前一段不知道的还以为我真是点出题意了，多歧亡羊的“亡”并非死亡，而是丢失，说的不过是某个小子因为岔路太多没找到自己丢的羊的事情，末了担忧一句，牧羊人多歧亡羊，读书人则患不专，“大道以多歧亡羊，学者以多方丧生”。正如前些年在学校论坛上就明白的东西，GNU/Linux 是拿来用的，不是拿来玩的。
